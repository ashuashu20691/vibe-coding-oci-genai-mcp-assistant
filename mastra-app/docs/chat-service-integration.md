# ChatService Event Integration

## Overview

The ChatService provides event-driven narrative streaming integration with Mastra's agent system. It coordinates tool execution with conversational explanations, creating a Claude Desktop-like experience where the AI explains its reasoning before and after every action.

## Key Features

### 1. Event Handler Registration (Requirement 17.4)

The ChatService allows registration of custom event handlers that are invoked at appropriate lifecycle points:

```typescript
chatService.registerEventHandlers({
  onToolCall: async function* (toolCall: ToolCall) {
    yield 'Custom pre-tool narrative';
  },
  
  onStepFinish: async function* (step: ProcessingStep) {
    yield 'Custom post-step narrative';
  },
  
  onError: async function* (error: Error) {
    yield 'Custom error narrative';
  },
});
```

### 2. Pre-Tool Narrative (Requirement 17.1)

Before each tool execution, the service streams a conversational explanation:

```typescript
// Input: Tool call to list_tables
// Output: "Let me check what tables are available..."
```

The narrative is generated by `NarrativeStreamingService.streamPreToolNarrative()` and streamed character-by-character for a smooth effect.

### 3. Post-Tool Narrative (Requirement 17.2)

After tool execution completes, the service interprets the results in natural language:

```typescript
// Input: Tool result with 3 tables
// Output: "I found 3 items from what tables are available. Let me analyze this data."
```

The narrative is generated by `NarrativeStreamingService.streamPostToolNarrative()` and provides context for the next action.

### 4. Error Narrative (Requirement 17.5)

When errors occur, the service explains what went wrong and what will be tried next:

```typescript
// Input: Connection error
// Output: "I'm having trouble connecting to the database. Let me try a different approach instead."
```

The narrative is generated by `NarrativeStreamingService.streamErrorNarrative()` and includes recovery plans.

### 5. Iteration Tracking

During autonomous retry loops, the service emits iteration updates:

```typescript
{
  type: 'iteration_update',
  iteration: 3,
  maxIterations: 5
}
```

This allows the UI to display progress indicators like "Working... (3/5)".

## Enhanced Stream Events

The ChatService emits enhanced stream events that include narrative content:

```typescript
type EnhancedStreamEvent = 
  | { type: 'narrative'; content: string }           // Conversational explanation
  | { type: 'content'; content: string }             // Assistant response
  | { type: 'tool_call'; toolCall: ToolCall; narrative: string }
  | { type: 'tool_result'; result: ToolResult; narrative: string }
  | { type: 'iteration_update'; iteration: number; maxIterations: number }
  | { type: 'error'; error: string; narrative: string }
  | { type: 'done'; finishReason: string };
```

## Usage Example

### Basic Usage

```typescript
import { createChatService } from '@/services/chat-service';
import { narrativeStreamingService } from '@/services/narrative-streaming-service';
import { OCIModelAdapter } from '@/mastra/providers/oci-model-adapter';

// Create model adapter
const modelAdapter = new OCIModelAdapter({
  provider: ociProvider,
  modelId: 'cohere.command-r-plus',
  conversationId: 'my-conversation',
});

// Create chat service
const chatService = createChatService(modelAdapter, narrativeStreamingService, {
  systemPrompt: 'You are a helpful assistant.',
  maxIterations: 5,
});

// Stream messages with narrative
for await (const event of chatService.sendMessageWithNarrative(messages)) {
  switch (event.type) {
    case 'narrative':
      console.log('[Narrative]', event.content);
      break;
    case 'content':
      console.log('[Content]', event.content);
      break;
    case 'tool_call':
      console.log('[Tool]', event.toolCall.name);
      break;
    case 'iteration_update':
      console.log(`[Progress] ${event.iteration}/${event.maxIterations}`);
      break;
  }
}
```

### API Route Integration

```typescript
export async function POST(request: NextRequest) {
  const { messages, modelId, conversationId } = await request.json();

  const modelAdapter = new OCIModelAdapter({
    provider: ociProvider,
    modelId,
    conversationId,
  });

  const chatService = createChatService(modelAdapter, narrativeStreamingService);

  const encoder = new TextEncoder();
  const stream = new ReadableStream({
    async start(controller) {
      for await (const event of chatService.sendMessageWithNarrative(messages)) {
        controller.enqueue(encoder.encode(`data: ${JSON.stringify(event)}\n\n`));
      }
      controller.close();
    },
  });

  return new Response(stream, {
    headers: {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache',
    },
  });
}
```

## Event Flow

### Successful Tool Execution

```
1. User submits message
2. ChatService receives message
3. Model generates tool call
4. ┌─ onToolCall event fires
5. │  └─ Pre-tool narrative streams: "Let me check..."
6. │  └─ tool_call event emitted
7. Tool executes
8. Tool returns result
9. ┌─ onStepFinish event fires (via callback)
10.│  └─ Post-tool narrative streams: "I found 3 items..."
11.│  └─ tool_result event emitted
12.Model generates response
13.└─ content events stream
14.done event emitted
```

### Error and Retry Flow

```
1. Tool execution fails
2. ┌─ onError event fires
3. │  └─ Error narrative streams: "That didn't work..."
4. │  └─ error event emitted
5. │  └─ iteration_update event emitted (1/5)
6. Model tries alternative approach
7. ┌─ onToolCall event fires
8. │  └─ Pre-tool narrative: "Let me try a different approach..."
9. Tool executes with different strategy
10.Success or continue retry loop
```
