# Agent Visualization Behavior Fix

## Issue
After fixing the technical visualization generation bug, the AI agent started apologizing and saying it "cannot create visuals, charts, or graphs directly" even though the system is fully capable of generating visualizations.

## Root Cause
The system prompts and agent instructions did not explicitly inform the AI that:
1. Visualizations are automatically generated by the system
2. The AI should never refuse to create visualizations
3. The AI just needs to execute queries and the system handles visualization generation

This caused the AI to incorrectly believe it lacked visualization capabilities, leading to apologetic responses and refusal to generate visuals.

## Solution
Updated system prompts and agent instructions to explicitly state visualization capabilities:

### 1. Database Agent Instructions (`src/mastra/agents/database-agent.ts`)
Added a new section:
```
VISUALIZATION CAPABILITIES (CRITICAL):
YOU CAN AND MUST CREATE VISUALIZATIONS! The system automatically generates and displays them.
NEVER say "I cannot create visuals/charts/graphs" - you absolutely can!

When query results are returned:
- The system AUTOMATICALLY generates visualizations (charts, tables, dashboards)
- These are displayed inline in the chat interface
- You don't need to do anything special - just execute the query
- The visualization generation happens automatically after the query completes

If a user asks for a specific visualization type (bar chart, pie chart, dashboard):
- Execute the query to get the data
- The system will automatically generate the requested visualization
- Simply confirm what you're showing them

NEVER tell users you cannot create visualizations - the system does this automatically!
```

### 2. Enhanced System Prompt (`src/services/system-prompts.ts`)
Added a prominent section at the top:
```
CRITICAL - VISUALIZATION CAPABILITIES:
YOU CAN AND DO CREATE VISUALIZATIONS! The system automatically generates them.
NEVER say "I cannot create visuals/charts/graphs/dashboards" - you absolutely can!
When you execute a query:
- The system AUTOMATICALLY generates visualizations from the results
- Charts, tables, and dashboards are displayed inline in the chat
- You don't need to do anything special - just execute the query
- Simply describe what the visualization shows after it's generated

If a user asks for a specific visualization:
- Execute the query to get the data
- The system will automatically generate the requested visualization type
- Confirm what you're showing them (e.g., "Here's the bar chart showing sales by region")
```

## Expected Behavior After Fix
1. When users ask for visualizations, the AI will:
   - Execute the appropriate SQL query
   - Confidently state what visualization is being generated
   - NOT apologize or claim inability to create visuals

2. The system will:
   - Automatically generate visualizations from query results
   - Display them inline in the chat interface
   - Support all visualization types (bar, line, pie, table, dashboard, etc.)

3. The AI will:
   - Describe what the visualization shows
   - Offer to create different visualization types if requested
   - Maintain confidence in its visualization capabilities

## Testing
To verify the fix works:
1. Start the development server
2. Ask the AI: "Show me sales data broken down by product category as a bar chart"
3. Expected: AI executes query and confidently presents the bar chart
4. NOT expected: AI apologizes or says it cannot create visualizations
